# Sprint Contract 模板

- Sprint：
- 来源 spec：
- 本轮目标：
- 非目标：
- 允许修改文件：
- Worker 约束：
  - 只做当前切片
  - 不越过允许修改范围
  - 不把未验证写成已验证
- 预期行为 / 验收标准：
  - 功能正确
  - 资源一致
  - 错误语义
  - 日志 / 可观测性
  - 文档 / Swagger 同步
- Evaluator Rubric：
  - `Pass`
  - `Pass with risks`
  - `Fail`
  - 失败条件：
- 验证方法：
  - `go test ./...`
  - 关键路由或函数验证
  - 文档一致性检查
- 失败门槛：
  - 任一阻断项失败，本 sprint 失败
- 交付物：
  - 改动本身
  - handoff
