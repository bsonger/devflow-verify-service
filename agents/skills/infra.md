# Infra Skill

## Purpose

定义 Kubernetes / Argo / Tekton 相关操作的默认安全边界。

## Scope

适用于：

- 集群状态读取
- 事件排查
- 基于清单和配置的分析

不适用于：

- 用户明确要求并授权的外部资源变更

## Must

- 所有集群相关操作默认只读
- 文档和结论中明确标注外部实际操作需要人工审核
- 对外部资源的假设必须显式写出

## Must Not

- 不执行创建、更新、删除集群资源的操作，除非明确被要求
- 不把未验证的集群推断写成既成事实

## Outputs

- 只读排查结果
- 明确的外部操作边界

## Pass/Fail

- `Pass`：外部系统保持只读，结论边界清晰
- `Fail`：越权变更集群资源或隐瞒外部假设
